System bankowy

1.Rejestracja  DONE

-wprowadzenie loginu oraz sprawdzenie czy nie został już taki zapisany DONE

-wprowadzenie hasła DONE

-wygenerowanie nr konta, pinu oraz unikalnego kodu DONE

2.Logowanie DONE

-wprowadzenie loginu i hasła DONE

-sprawdzenie czy login został już zapisany, jeśli tak sprawdzenie czy dla danego loginu zostało podane poprawnie hasło, jeśli tak to daj dostęp do konta, jeśli nie komunikat DONE

4. Edycja pinu DONE

-dla zalogowanego użytkownika jest dostępna edycja pinu DONE

-użytkownik musi podać swój pin, hasło oraz 2 razy nowy pin DONE

-jeśli pin i hasło się zgadzają następuje zmiana pinu przypisanego do danego konta DONE

5. Blokada karty DONE

-użytkownik podaje hasło, nr konta oraz pin DONE 

-jeśli informacje się zgadzają karta zostaje zablokowana DONE

-zablokowana karta uniemożliwa wpłaty i wypłaty NOT YET

6. Wpłaty i wypłaty pieniędzy DONE

-osoba musi podać pin oraz numer konta DONE

-możliwość wypłaty albo wpłaty pieniędzy DONE

-przy próbie wypłaty sumy większej niż na koncie komunikat DONE

7.Data ostatniego logowania DONE

-dla zalogowanych użytkowników jest dostępna opcja sprawdzenia daty ostatniego logowania DONE

-data przy każdym logowaniu jest aktualizowana DONE

8. Konto główne i oszczędnościowe 

-zalogowany użytkownik ma dostęp do dwóch  rodzai konta DONE

-główne odpowiedzialne za wpłaty/wypłaty oraz przelewy (zablokowano mozliwosc wplaty/wyplaty z konta oszczednosciowego z poziomu bankomatu) 50/50

-oszczędnościowe na które można przelać środki tylko z konta głównego oraz wypłacić środki tylko na główne konto

9.kalkulator lokaty DONE

-opcja dostępna dla wszystkich użytkowników DONE

-użytkownik podaje kapitał, czas lokaty, oprocentowanie oraz ilosć kapitalizacji w skali roku DONE

10.kalkulator walutowy DONE

-użytkownik może wybrać na jaką walutę chce przeliczyć swoje środki DONE

-program wyświetla w zaokrągleniu do 2 miejsc po przecinku wynik DONE

-kalkulator nie zmienia waluty na koncie DONE

11.Szybki przelew DONE

-zalogowany użytkownik podaje unikalny kod innego użytkownika DONE

-jeśli taki istnieje daje możliwość  podania sumy i przelania DONE

-jeśli nie istnieje użytkownik o takim kodzie  to wyświetla komunikat i daje możliwość wpisania ponownie unikalnego kodu DONE

-program sprawdza czy użytkownik ma daną sumę na koncie DONE

12. Tradycyjny przelew

-zalogowany użytkownik może wybrać z menu przelew tradycyjny

-podaje on wtedy nr konta innego użytkownika oraz kwotę

-jeśli posiada taką kwotę, co chce przelać przelew zostaje zatwierdzony
